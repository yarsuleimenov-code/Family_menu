# Rollback пилота

## Точки восстановления

- код до pilot deployment: `08da1c37358697f3a77b194fa9358a81ed727806`;
- backup Google Sheet: `PILOT_BACKUP Family Menu 2026-07-18 before 08da1c3`;
- предыдущая версия Apps Script остаётся в истории и управлении deployments;
- предыдущие frontend assets доступны через Git history.

## Когда откатывать

- приложение не загружает production-данные;
- сохранения системно завершаются ошибкой;
- обнаружена потеря или массовое изменение пользовательских данных;
- Blocker/Major воспроизводится после reload.

## Порядок

1. Остановить пилот и не выполнять новые mutations.
2. В Apps Script открыть `Deploy → Manage deployments` и вернуть предыдущую рабочую версию deployment.
3. Во frontend восстановить предыдущий commit, выполнить `pnpm run build:pages`, commit и push в `main`.
4. Если повреждены данные, не копировать отдельные строки вручную: сравнить production с backup и восстановить только подтверждённые диапазоны или целиком заменить файл после отдельного подтверждения владельца.
5. Выполнить read-only `getAppData`, затем один изолированный smoke с `PILOT_TEST` и очисткой.

Полное восстановление Google Sheet из backup является последним шагом: оно заменяет все изменения, сделанные после backup.
