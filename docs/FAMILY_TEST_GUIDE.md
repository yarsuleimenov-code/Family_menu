# Инструкция семейного теста

Цель теста: проверить, удобно ли использовать Family Menu в реальном недельном сценарии: выбрать ужины, собрать покупки и пройтись по списку в магазине.

## Перед началом

- Открыть приложение: https://yarsuleimenov-code.github.io/Family_menu/
- Использовать один основной телефон для проверки shopping list, чтобы локальные отметки `В корзине` не расходились между устройствами.
- Если приложение долго открывается, подождать первое обновление данных из Google Sheets. Повторные открытия должны быть быстрее за счёт локального кэша.

## Сценарий 1. Выбрать ужин на день

1. Открыть `План`.
2. Выбрать дату.
3. Выбрать блюдо из вариантов или нажать `Случайное блюдо`.
4. Дождаться статуса сохранения.
5. Если появилась ошибка сохранения, не обновлять страницу сразу: выбор остаётся локально, можно нажать `Повторить`.

Критерий успеха: понятно, какое блюдо выбрано на день, и нет сомнений, сохранился ли выбор.

## Сценарий 2. Собрать неделю

1. В `Плане` выбрать начало недели.
2. Проверить диапазон дат.
3. При необходимости нажать `Заполнить неделю случайно`.
4. Подтвердить действие в dialog.
5. Просмотреть недельный список и вручную заменить неподходящие дни.

Критерий успеха: меню на неделю собрано за 5-10 минут без повторов, которые сразу бросаются в глаза.

## Сценарий 3. Список покупок

1. Открыть `Покупки`.
2. Проверить даты списка.
3. Включить или выключить `Добавить базовые покупки`.
4. Нажать `Обновить список`.
5. Проверить сумму, количество товаров и товары без цены.
6. В магазине отмечать чекбокс `В корзине`.
7. Если товар уже есть дома, нажать `Дома`.
8. При необходимости использовать `Скрыть купленное`, `Скопировать`, `Очистить`.

Критерий успеха: список удобно использовать одной рукой, купленные товары не исчезают без контроля, отметки сохраняются после перезагрузки.

## Сценарий 4. База блюд

1. Открыть `Блюда`.
2. Найти блюдо поиском.
3. Проверить, что у активных блюд есть время, порции, ингредиенты и бюджет.
4. Если блюдо проблемное, открыть редактирование и заполнить недостающие поля.

Критерий успеха: random dinner выдаёт реально пригодные блюда, а покупки строятся без пустых ингредиентов.

## Сценарий 5. Базовые продукты

1. Открыть `Базовые`.
2. Проверить часто покупаемые продукты.
3. Заполнить цену, если её нет.
4. Отключить продукты, которые больше не нужно покупать регулярно.

Критерий успеха: базовые покупки не раздувают список лишними позициями, а сумма остаётся примерно полезной.

## Что фиксировать во время теста

- Где было непонятно, что нажимать.
- Где пришлось долго скроллить.
- Какие блюда не подходят семье.
- Какие товары попали в покупки ошибочно.
- Какие товары не объединились или имеют неудобные единицы.
- Где не хватило цены.
- Были ли сомнения, сохранилось ли действие.

## Известные ограничения тестовой версии

- Это личное приложение без полноценного multi-user режима.
- Отметки shopping list хранятся локально на устройстве.
- Apps Script может медленно отвечать после простоя.
- Цены ориентировочные.
- Ручные товары в текущем списке пока локальные и не являются полноценной базой продуктов.

## Когда тест считать успешным

- Ужин на день выбирается без объяснений.
- Неделя собирается за 5-10 минут.
- Список покупок можно использовать в магазине без бумажной копии.
- После 1-2 недель теста понятно, какие блюда и продукты нужно поправить в Google Sheet.
