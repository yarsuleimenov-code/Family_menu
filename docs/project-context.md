# Контекст продукта

## Пользователь и бизнес-вопрос

Основной пользователь — семья, которой нужно быстро выбрать ужины, получить объединённый список покупок и не потерять изменения при нестабильной связи.

## Основной сценарий

1. Выбрать или случайно подобрать блюдо на дату.
2. Проверить недельный план; случайный подбор исключает уже выбранные блюда и запрещённые продукты, ручной выбор требует проверки пользователя.
3. Сформировать покупки по выбранным блюдам; базовые продукты подключаются отдельно.
4. Отметить товары и сохранить сессию покупок.

## Критерии готовности v1

- Подтверждено кодом: React-клиент работает с mock/localStorage или Google Sheets через Apps Script API.
- Подтверждено сборкой 2026-07-18: TypeScript и production build проходят.
- Бизнес-требование: семейный тест должен подтвердить удобство и корректность живых данных.
- Открытый вопрос: какой уровень одновременной работы нескольких членов семьи требуется.
