## Что проверить вручную в семейном тесте

Автоматический аудит не может оценить вкусовую и бытовую пригодность. Перед первой полной неделей стоит вручную посмотреть:

- блюда, которые семья точно не будет готовить, и деактивировать их;
- `include_by_default` у базовых продуктов, чтобы список покупок не раздувался;
- цены после первого похода в магазин, особенно мясо, рыбу, молочные продукты и овощи;
- блюда с остатками: действительно ли хватает на обед на следующий день;
- блюда дольше 80-90 минут: оставить их для выходных или деактивировать;
- единицы измерения, которые неудобно покупать в магазине.

## Команды проверки

Read-only аудит:

```powershell
$env:APPS_SCRIPT_ENDPOINT="https://script.google.com/macros/s/.../exec"
$env:API_TOKEN=""
node scripts/audit_live_data.mjs
```

Backend validation:

```powershell
$env:APPS_SCRIPT_ENDPOINT="https://script.google.com/macros/s/.../exec"
$env:API_TOKEN=""
node -e "/* call validateData */"
```

## Ограничения аудита

- Аудит не меняет данные в Google Sheets.
- Аудит проверяет структурную пригодность данных, а не качество рецептов.
- Цены считаются пригодными, если заполнен `price_per_unit` или `estimated_package_price`.
- Запрещённые продукты ищутся по словам, чтобы `нут` не срабатывал внутри слова `минут`.
